## 📊 **3️⃣ FLEXIBILITÉ COMMERCIALE**

### Vue d'ensemble

L'architecture modulaire de Synd_Gest permet de **proposer différentes offres commerciales** adaptées aux besoins spécifiques de chaque type de client. Cette approche permet de maximiser le marché adressable et d'optimiser la tarification.

### 🏘️ **Extension 1 : RÉSIDENCES ÉTUDIANTES**

#### 📊 **Marché potentiel**
- **Taille du marché France** : ~170 000 logements étudiants privés
- **Acteurs majeurs** : Studefi, Les Estudines, Nexity Studéa, Cardinal Campus
- **Modèle économique** : Similaire à Domitys (propriétaires investisseurs + exploitant + locataires)

#### 🎯 **Adaptations spécifiques nécessaires**

##### Nouveau rôle : **ETUDIANT** (équivalent de RESIDENT)
```sql
CREATE TABLE etudiants (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    civilite ENUM('M', 'Mme'),
    nom VARCHAR(100),
    prenom VARCHAR(100),
    date_naissance DATE,
    telephone VARCHAR(20),
    email VARCHAR(150),
    -- Spécifique étudiants
    etablissement VARCHAR(200),  -- Université, école
    niveau_etudes ENUM('L1', 'L2', 'L3', 'M1', 'M2', 'Doctorat'),
    annee_scolaire VARCHAR(20),
    garant_nom VARCHAR(100),
    garant_telephone VARCHAR(20),
    garant_email VARCHAR(150),
    -- Caution
    depot_garantie DECIMAL(10,2),
    date_entree DATE,
    date_sortie_prevue DATE,
    actif BOOLEAN DEFAULT TRUE
);
```

##### Adaptations fonctionnelles
| Fonction Domitys | Adaptation Étudiants |
|------------------|----------------------|
| Âge minimum (60 ans) | Âge 18-30 ans |
| Niveau d'autonomie | Niveau d'études |
| Animations seniors | Événements étudiants, soirées |
| Restauration quotidienne | Cafétéria, distributeurs |
| Services médicaux | Infirmerie ponctuelle |
| Loyer mensuel élevé (1450€) | Loyer réduit (500-800€) |
| Baux longs (1-2 ans) | Baux 9-12 mois (année scolaire) |
| Portail famille | Portail étudiant + garant |

##### Dashboard exploitant étudiant
```
KPIs spécifiques:
• Taux d'occupation par période (sept-juin)
• Liste d'attente pour prochaine rentrée
• Pré-réservations étudiants étrangers
• Renouvellements confirmés
• Impayés étudiants (avec relance garant)
• Événements du mois
```

#### 💰 **Pricing spécifique**
```
FORMULE RÉSIDENCE ÉTUDIANTE
────────────────────────────
149€ HT/mois par résidence
+ 1,50€ HT/mois par logement

Exemple: Résidence de 120 studios
= 149€ + (120 × 1,50€) = 329€ HT/mois
```

#### 🎯 **Arguments de vente**
- ✅ Gestion des renouvellements annuels automatisée
- ✅ Interface garant pour suivi paiements
- ✅ Gestion des pré-réservations (rentrée)
- ✅ Calcul automatique APL/CAF
- ✅ Export pour déclarations fiscales

### 🏖️ **Extension 2 : RÉSIDENCES DE TOURISME**

#### 📊 **Marché potentiel**
- **Taille du marché** : ~2 000 résidences de tourisme en France
- **Acteurs** : Pierre & Vacances, Maeva, Lagrange, Odalys
- **Modèle** : LMNP (Loueur Meublé Non Professionnel)

#### 🎯 **Adaptations spécifiques**

##### Nouveau rôle : **TOURISTE** (location courte durée)
```sql
CREATE TABLE reservations_tourisme (
    id INT PRIMARY KEY AUTO_INCREMENT,
    lot_id INT,
    exploitant_id INT,
    -- Client
    client_nom VARCHAR(100),
    client_email VARCHAR(150),
    client_telephone VARCHAR(20),
    nb_adultes INT,
    nb_enfants INT,
    -- Séjour
    date_arrivee DATE,
    date_depart DATE,
    nb_nuits INT,
    prix_nuit DECIMAL(10,2),
    montant_total DECIMAL(10,2),
    acompte DECIMAL(10,2),
    -- Statut
    statut ENUM('reservation', 'confirmee', 'encours', 'terminee', 'annulee'),
    canal_reservation ENUM('site_web', 'booking', 'airbnb', 'telephone', 'autre')
);
```

##### Adaptations fonctionnelles
| Fonction Domitys | Adaptation Tourisme |
|------------------|---------------------|
| Loyer mensuel | Prix par nuit variable (saison) |
| Occupation longue durée | Séjours 1-14 jours |
| Résident permanent | Clients ponctuels |
| Services quotidiens | Services hôteliers |
| Contrat hébergement | Réservation en ligne |
| Portail famille | Portail client |
| Marge fixe mensuelle | Marge variable selon saison |

##### Dashboard exploitant tourisme
```
KPIs spécifiques:
• Taux d'occupation par saison
• RevPAR (Revenue Per Available Room)
• Tarif moyen par nuitée
• Réservations par canal (direct, OTA)
• Taux d'annulation
• Note moyenne clients (TripAdvisor, Booking)
• Calendrier des disponibilités
• Prévisions haute saison
```

#### 💰 **Pricing spécifique**
```
FORMULE RÉSIDENCE TOURISME
───────────────────────────
Base: 299€ HT/mois par résidence
+ 3€ HT/mois par logement
+ 0,50€ par réservation traitée

Exemple: 50 appartements, 200 réservations/mois
= 299€ + (50 × 3€) + (200 × 0,50€) = 549€ HT/mois
```

#### 🔗 **Intégrations nécessaires**
- 🔌 API Booking.com
- 🔌 API Airbnb
- 🔌 Channel Manager (Cubilis, RoomCloud)
- 🔌 Système de paiement en ligne (Stripe, PayPal)
- 🔌 Caisse hôtelière

### 🏢 **Extension 3 : COLIVING**

#### 📊 **Marché potentiel**
- **Tendance** : Marché en forte croissance (nomades digitaux, jeunes actifs)
- **Acteurs** : Colonies, Urban Campus, Luko, Sharies
- **Modèle** : Espaces partagés + chambres privées + services communautaires

#### 🎯 **Adaptations spécifiques**

##### Nouveau concept : **COLIVERS** + **ESPACES COMMUNS**
```sql
CREATE TABLE colivers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    nom VARCHAR(100),
    prenom VARCHAR(100),
    date_naissance DATE,
    -- Profil coliving
    profession VARCHAR(150),
    entreprise VARCHAR(150),
    teletravail BOOLEAN,
    centres_interet TEXT,  -- Sport, Tech, Art, Cuisine...
    -- Abonnement
    type_abonnement ENUM('mensuel', 'trimestriel', 'annuel', 'nomade'),
    date_debut DATE,
    date_fin DATE,
    -- Chambre privée
    chambre_id INT,
    -- Accès espaces communs
    acces_coworking BOOLEAN DEFAULT TRUE,
    acces_salle_sport BOOLEAN DEFAULT TRUE,
    acces_cuisine BOOLEAN DEFAULT TRUE,
    acces_terrasse BOOLEAN DEFAULT TRUE
);

CREATE TABLE espaces_communs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    residence_id INT,
    type ENUM('coworking', 'cuisine', 'salon', 'salle_sport', 'salle_cinema', 'terrasse'),
    nom VARCHAR(100),
    surface INT,
    capacite INT,
    equipements TEXT,
    reservation_requise BOOLEAN DEFAULT FALSE
);

CREATE TABLE reservations_espaces (
    id INT PRIMARY KEY AUTO_INCREMENT,
    espace_id INT,
    coliver_id INT,
    date_reservation DATE,
    heure_debut TIME,
    heure_fin TIME,
    nb_personnes INT,
    statut ENUM('confirmee', 'annulee')
);
```

##### Adaptations fonctionnelles
| Fonction Domitys | Adaptation Coliving |
|------------------|---------------------|
| Résidents seniors | Jeunes actifs 25-40 ans |
| Animations seniors | Événements networking, workshops |
| Loyer fixe | Abonnements flexibles (1-12 mois) |
| Services médicaux | Services pro (coworking, wifi pro) |
| Occupation par lot | Chambre + accès espaces communs |
| Portail famille | Portail communautaire |

##### Dashboard exploitant coliving
```
KPIs spécifiques:
• Taux d'occupation chambres
• Taux d'utilisation espaces communs
• Durée moyenne de séjour
• Renouvellements d'abonnement
• Événements du mois
• Score satisfaction communauté
• Revenus services annexes (coworking journée)
```

#### 💰 **Pricing spécifique**
```
FORMULE COLIVING
─────────────────
249€ HT/mois par résidence
+ 2,50€ HT/mois par chambre

Exemple: 40 chambres
= 249€ + (40 × 2,50€) = 349€ HT/mois
```
